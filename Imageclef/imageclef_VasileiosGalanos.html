ImageCLEF Medical Caption Task 2019¶

The Image CLEF 2019 Concept Detection Task is a large-scale multi-label classification task aiming to identify medical terms (concepts) in radiology images. We will implement a system to classify a medical image based on several abnormalities represented by Unified Medical Language System (UMLS) concept IDs.

def list_files(start_path, num_files):
    for root, dirs, files in os.walk(start_path):
        level = root.replace(start_path, '').count(os.sep)
        indent = ' ' * 4 * (level)
        print(indent + os.path.basename(root) + '/', end='')
        if len(files) != 0:
            print()
        sub_indent = ' ' * 4 * (level + 1)
        print(sub_indent, end='')
        for i, f in enumerate(files):
            print(f, end=' ')
            if i == num_files - 1:
                break
        print()

Obviously, we cannot expect to load all of our images to memory in order to train our models due to the large storage space that it would need.


    

  • Thus, we use an ImageDataGenerator, offerred by keras, in order to read the images in batches.
    • 

  • This will create a Python generator that, when we call it, will return the next batch and also scale the pixel value from 0 to 1.
    • 

  • We use this generator with flow_from_dataframe method in order to read the images from the directory according to the values of the training set dataframe df
    • 

  • Each image will be resized to IMG_HEIGHT * IMAGE_WIDTH and the contents of the dataframe will be shuffled
    •

In order to find the most common tags,


    

  • we use a CountVectorizer fitted in all the concepts from the file string_concepts.csv
    • 

  • we then create a list with all the available tags from the training and validation set
    • 

  • we transform this list using the vectorizer, in order to produce a bag of words. Now for each tag, we have the number of times that it appeared in all the datasets.
    • 

  • We then transform the frequencies to probabilities using the total number of tags that appeared in both datasets.
    • 














In [16]:



    


from sklearn.feature_extraction.text import CountVectorizer

vectorizer = CountVectorizer(lowercase=False)

# fit vectorizer to unique terms from string concepts
vectorizer.fit(cn.tag_id.values.tolist())

# flatten the list of lists
train_captions_list = [labels[item] for sublist in train_data_gen.classes for item in sublist]
val_captions_list = [labels_val[item] for sublist in val_data_gen.classes for item in sublist]
captions_list = train_captions_list + val_captions_list

# keep total number of captions in training set
total_captions = len(captions_list) #len(set(train_captions_list))

# transform the input for vectorizer
joined_captions_list = [','.join(captions_list)]

# apply the vectorizer
captions_count = vectorizer.transform(joined_captions_list).todense()

# create dataframe using the vectorizer's vocabulary
captions_freqs = pd.DataFrame.from_dict(vectorizer.vocabulary_,orient='index',columns=['ind'])

# create column with the frequencies for each caption in the training set
captions_freqs['freq'] = captions_freqs['ind'].apply(lambda x: captions_count.item(x))

# create column with probabilities using unique training captions
captions_freqs['prob'] = captions_freqs['freq']/total_captions

captions_freqs = captions_freqs.sort_values(by=['freq'],ascending=False)
print('Total number of tags',total_captions)
print('Most frequent tags :\n\n', captions_freqs.iloc[:10,2])

Baseline Model¶
We create a baseline classifier in order to measure our efforts.


    

  • This classifier, receives a list with the image filenames , and a seed for the random generator.
    • 

  • It then samples from all the available tags, using their corresponding probabilities tha we found earlier
    • 

  • It returns a dictionary with the filename and the six sampled tag ids.
    • 

  • We choose six, because it is the average number of tags per image.
    • 

  • Lastly, we create two files with the image names and their corresponding labels. val_baseline.csv stores the predicted labels using the baseline classifier on the validation set, while  val_ground_truth.csv is the ground truth file with the correct concept labels for each image in the validation set.
    • 














In [19]:



    


import csv

def baseline_classifier(images, input_seed):
    np.random.seed(seed)
    d = dict()
    for image in images:
        sampled_captions = list(np.random.choice(captions_freqs.index.values, 6, p=captions_freqs.prob.values,replace =False))
        d.update({image:sampled_captions})
    return d

baseline_val_dict = baseline_classifier(df_val.image.tolist(),1)
df_val['baseline_tags'] = df_val['image'].map(baseline_val_dict)

df_val['baseline_tags_formated'] = df_val['baseline_tags'].apply(lambda x: ';'.join(map(str,x)))
df_val['tags_formated'] = df_val['tags'].apply(lambda x: ';'.join(map(str,x)))

df_val.to_csv(r'val_baseline.csv'
              ,index=False
              ,header=False
              ,quoting=csv.QUOTE_NONE
              ,escapechar = ''
              ,sep='\t'
             ,columns=['image','baseline_tags_formated'])

df_val.to_csv(r'val_ground_truth.csv'
              ,index=False
              ,header=False
              ,quoting=csv.QUOTE_NONE
              ,escapechar = ''
              ,sep='\t'
             ,columns=['image','tags_formated'])

df_val.drop(['baseline_tags_formated', 'tags_formated'], axis=1, inplace=True)

The method that we will use in order to predict the concepts for the images is Transfer Learning.


    

  • We will use pre-trained deep learning models from Keras Applications, in order to extract the features from the images.
    • 

  • We will add the predictions layers for our multi-label classification problem accordingly, and train the models in our datasets only on those layers, because we don't want to change the way that these powerfull models extract features.
    • 

  • The first deep learning model is MobileNet V2, that we will compile with a binary crossentropy loss function and an Adam optimizer.
    • 

  • The last prediction layer will have as many outputs as our labels (5528) and a sigmoid activation function. We use this type of activation function (and not a softmax) because we have a multi-label classification problem where all our outputs are independent of each other.
    • 

  • We choose a batch size = 13 and resize our images to 128*128 in order to avoid to run out of RAM memory.
    •
